## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

> Rationale: Professional data-dense tool (Linear/Notion aesthetic per CONTEXT.md). 14px body maximises table row density for transaction lists. 12px labels for table headers and sidebar nav items. Two weights only (400 and 600 — medium 500 for labels only as a label-specific exception within the 2-weight system).

Font family: `Inter, ui-sans-serif, system-ui, -apple-system, sans-serif`

## Layout Contract

### App Shell

```
┌─────────────────────────────────────────────────────┐
│  Sidebar (240px expanded / 52px collapsed)          │
│  ┌─────────────────┐  ┌──────────────────────────┐  │
│  │ App logo/name   │  │  Main content area        │  │
│  │─────────────────│  │  ┌────────────────────┐   │  │
│  │ Company list    │  │  │ Page header        │   │  │
│  │  • Company A ●  │  │  │ (Transacções)      │   │  │
│  │  • Company B ○  │  │  ├────────────────────┤   │  │
│  │  • Company C ○  │  │  │ Filter bar         │   │  │
│  │─────────────────│  │  │ (search + date)    │   │  │
│  │ Nav             │  │  ├────────────────────┤   │  │
│  │  Transacções ✓  │  │  │ Data table         │   │  │
│  │─────────────────│  │  │ (skeleton or rows) │   │  │
│  │ Sage: Ligado ●  │  │  ├────────────────────┤   │  │
│  └─────────────────┘  │  │ Pagination         │   │  │
│                        │  └────────────────────┘   │  │
│                        └──────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

- Sidebar width: 240px expanded, 52px collapsed (icon-only)
- Collapse toggle: icon button at sidebar top-right, 44px touch target
- Sidebar collapse animation: 200ms ease-out width transition
- Active company: highlighted row with accent background, name in semibold, green dot for connected
- Inactive company: neutral row, name in regular weight, gray dot for not connected
- Sage status footer: sticky at sidebar bottom, always visible, "Sage: Ligado" green or "Sage: Desligado" gray

> Source: CONTEXT.md — "Sidebar + content layout"; "Collapsible to icons only"; "Sage connection status indicator at sidebar footer"

### Transactions Table

- Column order: Data | Referência | Descrição | Tipo | Valor
- "Valor" (amount) right-aligned, Portuguese format: `1.234,56 €`
- Sortable columns: click header to toggle asc/desc, active sort column shows directional arrow indicator
- Skeleton state: 8 animated placeholder rows matching table column layout
- Error state: full-width error card inside the table area (not a modal), with heading, body, and "Tentar novamente" button; "Detalhes" toggle below with COM error code/message
- Empty state (no data after filter): centered illustration-free message with heading and body copy
- Server-side pagination: 50 rows per page, page controls at table bottom (Anterior / Seguinte + page indicator)

> Source: CONTEXT.md — "Phase 1 shows transactions list only (date, reference, description, amount, type)"; "Skeleton table while Sage data is loading"; "Inline error with retry button"; "Server-side pagination, 50 rows per page"

## Accessibility

- Language attribute: `<html lang="pt">` (Portuguese)
- All interactive elements: keyboard navigable, focus-visible ring (2px offset)
- Color contrast: minimum 4.5:1 for body text on all surfaces (both light and dark modes)
- Status dots (green/gray): accompanied by text label "Sage: Ligado" / "Sage: Desligado" — not color-alone communication
- Skeleton rows: `aria-busy="true"` on the table container while loading
- Error region: `role="alert"` so screen readers announce it when it appears
