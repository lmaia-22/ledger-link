## Decisions Made

- **Manual components.json for monorepo:** The shadcn CLI installed components to `@/components` resolved from the project root (creating `/@/components/ui/`), not from the client directory. Files were moved manually to `client/src/components/ui/`.
- **Explicit routesDirectory in vite.config.ts:** TanStack Router Vite plugin defaulted to scanning `[projectRoot]/src/routes`, but routes live in `client/src/routes`. Fixed with `path.resolve(__dirname, './src/routes')` in plugin config.
- **Vite root set to `__dirname`:** Without explicit `root`, Vite couldn't find `index.html` when invoked from the project root with `--config client/vite.config.ts`.
